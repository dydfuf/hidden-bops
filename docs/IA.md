# Information Architecture (IA)

## AI 뮤직비디오 제작 플랫폼 "MelodyVision"

**버전:** 1.0  
**작성일:** 2025년 11월 23일  
**문서 유형:** Information Architecture & Site Map

***

## 1. IA 개요

### 1.1 목적

이 문서는 MelodyVision의 전체 정보 구조를 정의하며, 다음을 포함합니다:
- 사이트맵 및 페이지 계층구조
- 주요 사용자 플로우
- 네비게이션 시스템
- 콘텐츠 구조 및 분류 체계
- URL 구조

### 1.2 설계 원칙

1. **단순성**: 3-클릭 룰 - 모든 핵심 기능은 3번의 클릭 내 도달
2. **직관성**: 음악 용어 중심의 명명 체계
3. **확장성**: 향후 기능 추가를 고려한 구조
4. **일관성**: 전체 플랫폼에서 통일된 네비게이션 패턴
5. **접근성**: 명확한 정보 계층과 레이블링

## 3. 주요 사용자 플로우별 IA

### 3.1 첫 뮤직비디오 생성 플로우

```
/ (홈)
  ↓ [무료로 시작하기]
/signup
  ↓ [회원가입 완료]
/dashboard
  ↓ [+ 새 프로젝트]
/projects/new
  ↓ Step 1: 음악 업로드
/projects/new?step=1
  ↓ [음악 업로드 완료]
/projects/new?step=2
  ↓ Step 2: 캐릭터 설정
  │   ├─ 기존 캐릭터 선택 OR
  │   └─ [새 캐릭터 만들기] → /characters/new
  ↓ [캐릭터 선택 완료]
/projects/new?step=3
  ↓ Step 3: 템플릿 선택
  │   ├─ 템플릿 갤러리 보기 → /templates
  │   └─ 템플릿 선택
  ↓ [템플릿 선택 완료]
/projects/:id/edit
  ↓ Step 4: 씬 자동 생성 및 편집
  │   ├─ 씬 미리보기
  │   ├─ 개별 씬 편집 (모달)
  │   └─ 만족 시 진행
  ↓ [비디오 생성 시작]
/projects/:id/generate
  ↓ 생성 진행 중 (모니터링)
  ↓ [생성 완료]
/projects/:id/result
  ↓ 비디오 확인 및 다운로드
```

### 3.2 씬 편집 플로우

```
/projects/:id/edit (씬 편집 작업 공간)
  ↓
[씬 선택]
  ↓
씬 편집 모달 오픈 (Overlay)
  ├── 현재 이미지 표시
  ├── 설명 텍스트 편집
  ├── 빠른 옵션 선택
  │   ├─ 배경
  │   ├─ 시간대
  │   ├─ 카메라 앵글
  │   └─ 조명
  ├── [재생성] 버튼
  └── [저장] / [취소]
  ↓
[재생성 클릭]
  ↓
로딩 (2-3초)
  ↓
새 이미지 표시
  ↓
만족 시 [저장]
  ↓
모달 닫기
  ↓
/projects/:id/edit (업데이트된 씬)
```

### 3.3 결제 및 구독 플로우

```
/dashboard (무료 사용자)
  ↓ [플랜 업그레이드] 또는 한도 초과 시
/billing/plans
  ↓ 플랜 비교 및 선택
  ├─ 인디 뮤지션 ($29/월)
  ├─ 프로 아티스트 ($99/월)
  └─ 레이블/팀 ($299/월)
  ↓ [플랜 선택]
/billing/upgrade?plan=indie
  ↓ 결제 정보 입력 (Stripe)
  │   ├─ 카드 정보
  │   ├─ 청구지 주소
  │   └─ 약관 동의
  ↓ [결제 완료]
/billing?success=true
  ↓ 성공 메시지 표시
  ↓ 자동 리디렉션 (3초)
/dashboard
  ↓ 플랜 활성화 확인
```

### 3.4 캐릭터 프로필 생성 플로우

```
/dashboard 또는 /characters
  ↓ [새 캐릭터 만들기]
/characters/new
  ↓ Step 1: 사진 업로드
  │   ├─ 드래그 앤 드롭
  │   ├─ 파일 선택
  │   └─ 카메라 직접 촬영 (모바일)
  ↓ [2-3장 업로드 완료]
  ↓ Step 2: 자동 분석 (30초)
  │   ├─ 얼굴 감지
  │   ├─ 특징 추출
  │   └─ 캐릭터 프로필 생성
  ↓ Step 3: 테스트 이미지 생성
  │   ├─ 3-5장 자동 생성
  │   └─ 다양한 각도/표정
  ↓ Step 4: 확인 및 승인
  │   ├─ 만족 → [승인]
  │   └─ 불만족 → [재생성]
  ↓ [승인 완료]
/characters/:id
  ↓ 캐릭터 프로필 상세 페이지
  ↓ [이 캐릭터로 프로젝트 시작]
/projects/new?characterId=:id
```

## 4. 네비게이션 시스템

### 4.1 글로벌 네비게이션 (Top Navigation)

```
┌────────────────────────────────────────────────────────────┐
│ [Logo] MelodyVision    [대시보드] [프로젝트] [템플릿]       │
│                                     [🔔] [👤] [⚙️]         │
└────────────────────────────────────────────────────────────┘
```

**구성 요소:**

1. **왼쪽 영역**
   - 로고 (클릭 시 /dashboard)
   - 주요 메뉴:
     - 대시보드 (/dashboard)
     - 프로젝트 (/projects)
     - 템플릿 (/templates)

2. **오른쪽 영역**
   - 알림 아이콘 (🔔)
     - 드롭다운: 최근 알림 5개
     - "모두 보기" → /notifications
   - 프로필 아이콘 (👤)
     - 드롭다운 메뉴:
       - 내 프로필 → /settings/profile
       - 캐릭터 관리 → /characters
       - 결제 및 구독 → /billing
       - 설정 → /settings
       - 도움말 → /help
       - 로그아웃
   - 설정 아이콘 (⚙️) → /settings

### 4.2 사이드바 네비게이션 (작업 공간용)

프로젝트 편집 화면(/projects/:id/edit)에서 사용:

```
┌──────────────┐
│ 📊 개요       │
│ ✏️ 씬 편집    │ ← 현재 페이지
│ 🎬 비디오 생성│
│ 📈 결과       │
│ ⚙️ 설정       │
└──────────────┘
```

### 4.3 브레드크럼 (Breadcrumb)

```
홈 > 프로젝트 > Summer Night > 씬 편집
```

**패턴:**
- 모든 서브 페이지에 표시
- 클릭 가능한 링크
- 현재 페이지는 굵게 표시

### 4.4 탭 네비게이션 (Tab Navigation)

템플릿 갤러리(/templates)에서 사용:

```
┌─────────────────────────────────────────────┐
│ [전체] [장르별] [분위기별] [인기순] [최신순] │
└─────────────────────────────────────────────┘
```

## 5. 콘텐츠 구조 및 분류 체계

### 5.1 프로젝트 분류

**상태별:**
- 초안 (Draft)
- 편집 중 (Editing)
- 생성 중 (Generating)
- 완성 (Completed)
- 오류 (Error)
- 보관함 (Archived)

**정렬 옵션:**
- 최근 수정일
- 생성일
- 제목 (가나다순)
- 음악 길이

**필터:**
- 캐릭터별
- 템플릿별
- 날짜 범위
- 상태별

### 5.2 템플릿 분류

**장르별:**
- 팝 (Pop)
- 록 (Rock)
- 힙합 (Hip-hop)
- 발라드 (Ballad)
- 일렉트로닉 (Electronic)
- 인디 (Indie)
- 재즈 (Jazz)
- 기타 (Other)

**분위기별:**
- 밝고 경쾌한 (Bright & Cheerful)
- 차분하고 감성적 (Calm & Emotional)
- 어둡고 드라마틱 (Dark & Dramatic)
- 몽환적인 (Dreamy)
- 레트로 (Retro)
- 미니멀 (Minimal)
- 화려한 (Vibrant)

**스타일별:**
- 도시 (Urban)
- 자연 (Nature)
- 스튜디오 (Studio)
- 추상적 (Abstract)
- 애니메이션 (Animation)
- 일러스트 (Illustration)

## 6. URL 구조 및 명명 규칙

### 6.1 URL 패턴

**일관된 패턴:**
```
/{resource}                    - 리소스 목록
/{resource}/new                - 새 리소스 생성
/{resource}/:id                - 개별 리소스 조회
/{resource}/:id/edit           - 리소스 편집
/{resource}/:id/{sub-resource} - 서브 리소스
```

### 6.2 주요 URL 예시

**프로젝트:**
```
/projects                          - 프로젝트 목록
/projects/new                      - 새 프로젝트
/projects/abc123                   - 프로젝트 상세
/projects/abc123/edit              - 씬 편집
/projects/abc123/generate          - 비디오 생성
/projects/abc123/result            - 결과 확인
```

**캐릭터:**
```
/characters                        - 캐릭터 목록
/characters/new                    - 새 캐릭터 생성
/characters/xyz789                 - 캐릭터 상세
/characters/xyz789/edit            - 캐릭터 수정
/characters/xyz789/projects        - 사용된 프로젝트
```

**템플릿:**
```
/templates                         - 템플릿 갤러리
/templates?genre=pop               - 장르 필터
/templates?mood=dreamy             - 분위기 필터
/templates/urban-night             - 템플릿 상세
```

**설정:**
```
/settings                          - 설정 홈 (→ /settings/profile)
/settings/profile                  - 프로필 설정
/settings/account                  - 계정 설정
/settings/preferences              - 환경 설정
/settings/team                     - 팀 관리
```

### 6.3 쿼리 파라미터 규칙

**필터:**
```
/projects?status=completed         - 상태 필터
/projects?character=xyz789         - 캐릭터 필터
/templates?genre=pop&mood=bright   - 다중 필터
```

**정렬:**
```
/projects?sort=updated_desc        - 최근 수정일 내림차순
/projects?sort=title_asc           - 제목 오름차순
```

**페이지네이션:**
```
/projects?page=2                   - 2페이지
/projects?page=2&limit=20          - 페이지당 20개
```

**검색:**
```
/projects?q=summer                 - 검색 쿼리
/templates?search=urban            - 템플릿 검색
```

## 11. 정보 계층 (Information Hierarchy)

### 11.1 홈페이지 (Landing Page) 구조

```
Level 1: Hero Section
  ├── 메인 헤드라인
  ├── 서브 헤드라인
  ├── CTA 버튼 (무료로 시작하기)
  └── 데모 비디오

Level 2: 핵심 가치
  ├── 빠른 제작 (아이콘 + 설명)
  ├── 저렴한 비용
  └── 전문가 품질

Level 3: 작동 방식
  ├── Step 1: 음악 업로드
  ├── Step 2: 캐릭터 설정
  └── Step 3: AI가 자동 생성

Level 4: 작품 갤러리
  └── 사용자 제작 뮤직비디오 샘플

Level 5: 가격 안내
  └── 3개 플랜 비교 카드

Level 6: 고객 후기
  └── 실제 사용자 리뷰

Level 7: FAQ
  └── 자주 묻는 질문 5개

Level 8: CTA
  └── 재차 가입 유도
```

### 11.2 대시보드 정보 계층

```
Level 1: 환영 메시지 + 빠른 시작
  └── "새 프로젝트 시작하기" CTA

Level 2: 최근 프로젝트 (3개)
  ├── 썸네일
  ├── 제목
  ├── 상태
  └── 수정일

Level 3: 캐릭터 프로필 (2-3개)
  ├── 아바타
  ├── 이름
  └── "새 캐릭터 만들기"

Level 4: 사용량 통계
  ├── 진행률 바
  ├── 이번 달 사용량
  └── 플랜 정보

Level 5: 빠른 링크
  ├── 템플릿 둘러보기
  ├── 튜토리얼 보기
  └── 도움말
```

### 11.3 프로젝트 편집 페이지 계층

```
Level 1: 프로젝트 제목 + 액션 버튼
  ├── 프로젝트 이름
  ├── [저장]
  └── [비디오 생성]

Level 2: 타임라인 (시각적 개요)
  └── 전체 씬 타임라인

Level 3: 씬 리스트 (메인 콘텐츠)
  ├── 씬 1 카드
  ├── 씬 2 카드
  └── ...

Level 4: 씬 카드 세부 정보
  ├── 이미지 썸네일
  ├── 타임스탬프
  ├── 설명
  └── 액션 버튼 (재생성, 편집, 삭제)

Level 5: 사이드바 (2차 정보)
  ├── 프로젝트 정보
  ├── 캐릭터 정보
  └── 음악 정보
```

- [ ] 권한별 접근 제어가 명확한가?

### 14.3 접근성 체크리스트

- [ ] 키보드만으로 모든 페이지 탐색이 가능한가?

- [ ] 명확한 포커스 인디케이터가 있는가?
- [ ] 색상만으로 정보를 전달하지 않는가?
